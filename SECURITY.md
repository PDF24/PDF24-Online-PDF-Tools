# Security Policy

## Reporting a Vulnerability

We take the security of our project seriously. If you believe you have found a security vulnerability, please report it to us as soon as possible.

**Please do not report security vulnerabilities through public GitHub issues.**

### How to report
To report a vulnerability, please send an email to **[team@pdf24.org](mailto:team@pdf24.org)**.

To help us address the issue quickly, please include:
- A description of the vulnerability.
- Steps to reproduce (proof-of-concept scripts or screenshots).
- Potential impact.

### What to expect
- You will receive an acknowledgment of your report within a reasonable timeframe.
- We will provide a timeline for a fix and keep you updated on our progress.

## Disclosure Policy
We follow a coordinated disclosure model. We ask that you give us a reasonable amount of time to remediate the vulnerability before making any information public.
